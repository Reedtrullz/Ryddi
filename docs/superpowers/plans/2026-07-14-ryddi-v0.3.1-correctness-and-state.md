`BoundedFileTree` stores nodes with parent index, scope index, absolute depth, resource metadata, and a fixed `measurementDepth + 1` array of per-relative-depth measurements. A reverse in-memory pass shifts each child's layer by one into its parent. No finding may start another filesystem enumerator.

- [x] **Step 4: Implement this exact traversal policy**

```text
1. Validate every requested root and create one FIFO frontier per readable scope.
2. Visit one queued node per scope per round; never drain one scope first.
3. Carry integer depth with every queued URL.
4. Traverse no deeper than maximumFindingDepth + measurementDepth.
5. Do not follow symbolic links, package descendants, excluded paths, or unreadable descendants.
6. Record hard-link identity once in the shared scan context.
7. Stop scheduling descendants when the item budget or cancellation token is reached.
8. Reverse the captured node array and aggregate fixed depth layers in memory.
9. Materialize findings only through maximumFindingDepth, using each node's bounded aggregate.
```
