# Ryddi v0.3 Scan Session And Action Center Implementation Plan

> **For Reidar:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement this plan task-by-task.

**Goal:** Make Ryddi stateful and guided. Every scan should become a `ScanSession` with a clear lifecycle, durable evidence, invalidation rules, and an `ActionCenterReport` that tells the user the safest next step.

**Architecture:** Implement in `ReclaimerCore` first, expose through `reclaimer`, then render in SwiftUI. App views must not infer safety from strings or UI state; they consume core reports.

## Public Interfaces

Add:

```swift
public enum ScanSessionStage: String, Codable, Hashable, Sendable {
    case notStarted
    case scanned
    case reviewed
    case planReady
    case dryRunReady
    case reclaimReady
    case executed
    case recoveryAvailable
    case invalidated
}

public enum ScanSessionInvalidationReason: String, Codable, Hashable, Sendable {
    case rootsChanged
    case rulesChanged
    case policyChanged
    case findingsChanged
    case planSelectionChanged
    case receiptExpired
    case appVersionChanged
}

public struct ScanSession: Codable, Identifiable, Hashable, Sendable {
    public let id: String
    public let createdAt: Date
    public let updatedAt: Date
    public let appVersion: String
    public let ruleVersion: String
    public let preset: ScanPreset
    public let scopeDigest: String
    public let findingDigest: String?
    public let planDigest: String?
    public let dryRunReceiptID: String?
    public let executionReceiptID: String?
    public let stage: ScanSessionStage
    public let invalidationReasons: [ScanSessionInvalidationReason]
}

public enum ActionCenterItemKind: String, Codable, Hashable, Sendable {
    case grantAccess
    case runScan
    case reviewQueue
    case createPlan
    case runDryRun
    case executeSafePlan
    case quitApp
    case useNativeTool
    case reviewInFinder
    case exportReport
    case recoverItem
    case inspectRemoteReport
}

public enum ActionCenterPriority: String, Codable, Hashable, Sendable {
    case urgent
    case recommended
    case optional
    case blocked
}

public struct ActionCenterItem: Codable, Identifiable, Hashable, Sendable {
    public let id: String
    public let kind: ActionCenterItemKind
    public let priority: ActionCenterPriority
    public let title: String
    public let detail: String
    public let bytes: Int64?
    public let count: Int
    public let command: String?
    public let blockedReason: String?
    public let relatedQueue: ReviewQueueKind?
}

public struct ActionCenterReport: Codable, Hashable, Sendable {
    public let generatedAt: Date
    public let session: ScanSession?
    public let primaryAction: ActionCenterItem?
    public let items: [ActionCenterItem]
    public let nonClaims: [String]
}
```

The implementation may use existing model names when repo types differ, but the JSON shape must remain explicit and additive.
