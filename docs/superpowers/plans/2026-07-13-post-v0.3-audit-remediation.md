# Ryddi Post-v0.3 Audit Remediation Plan

**Goal:** Close every confirmed post-v0.3 audit finding without weakening Ryddi's report-first, recoverable, local-only safety model.

**Release posture:** Implement on `fix/post-v0.3-audit-remediation`. Do not publish `v0.3.1` in this plan. A patch release becomes eligible only after the signed workflow is provenance-bound, credentials are cleaned up, archive checksums verify portably, and the existing notarization/Gatekeeper gates pass.

## Task 1: Portable release integrity

- Add script tests that generate a staged fixture and prove both checksum files pass `shasum -c` from a different directory.
- Emit external checksums with only the archive basename.
- Replace the misleading `Ryddi.app` checksum entry with a deterministic payload archive entry that actually exists in the staged release.
- Extend manifest path-leak checks to checksum files.
- Correct README verification instructions and release evidence wording.

## Task 2: Signed-source provenance and secret cleanup

- Require a clean worktree for signed release gates.
- Require signed workflow inputs for version, build, and source ref; verify the checked-out SHA and tag/version relationship before importing credentials.
- Capture and restore the runner's previous default keychain.
- Delete the imported P12 and temporary keychain in an `if: always()` cleanup step.
- Pin GitHub Actions to reviewed immutable commit SHAs and retain comments naming their upstream release tags.
- Add source-shape/script tests for every fail-closed requirement.

## Task 3: Native executable identity binding

- Add a fakeable `ToolExecutableResolving` boundary that resolves a command to an absolute regular executable under approved roots.
- Add stable executable identity evidence based on absolute path plus filesystem identity and metadata.
- Resolve before native preview, mint the identity into the one-use capability, and resolve/revalidate immediately before perform.
- Launch native tools using their absolute executable URL, never `/usr/bin/env`.
- Keep read-only inventory/SSH/runtime probes compatible while preventing native perform through inherited `PATH`.
- Add tests for shadowed commands, path changes, replaced binaries, non-absolute execution, and successful stable preview/perform.

## Task 4: Explicit bounded history retention

- Remove the misleading ignored `keepLimit` save parameter and add explicit dry-run-first scan-history retention with fail-closed regular-file, identity, and symlink checks.
- Enable confirmed audit pruning only for plan-bound known JSON files after root containment, filename, regular-file, symlink, and filesystem-identity revalidation.
- Keep dry-run as the default; never schedule pruning.
- Use private directory/file permissions for newly written local evidence.
- Add tests for keep limits, replacement races, symlinks, unknown files, directories, legacy candidates, and receipt path redaction.

## Task 5: Real app behavior coverage

- Move live permission refresh state transition into a small core/app model with injectable probes.
- Test degraded-to-complete and complete-to-degraded transitions without source-string assertions.
- Keep packaged CLI fixture coverage, but add a deterministic app-model behavior suite to ordinary CI.
- Preserve Accessibility packaged E2E as the signed release proof path.

## Task 6: Remove stale ReclaimerAgent path

- Remove the unused `ReclaimerAgent` product, target, package payload, release assertions, and documentation.
- Verify LaunchAgent scheduling continues to invoke the report-only `reclaimer` CLI.
- Add a packaging regression proving no second scheduler executable ships.

## Verification

- `df -h /System/Volumes/Data`
- Focused tests for release scripts, native execution, audit retention, scan history, permission state, scheduler, and packaging.
- `swift test --scratch-path "$PWD/.build"`
- `swift build --scratch-path "$PWD/.build" -Xswiftc -warnings-as-errors`
- `bash -n Scripts/*.sh`
- `Scripts/release-check.sh`
- Verify generated checksum files from a clean external directory.
- `git diff --check` and clean temp review.

## Non-goals

- No new destructive cleanup classes.
- No automatic audit/history pruning.
- No remote cleanup execution, sudo management, root helper, telemetry, or path upload.
- No signing, notarization submission, GitHub release, or installed-app replacement during implementation.
