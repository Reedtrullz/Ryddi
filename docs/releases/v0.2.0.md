# Ryddi v0.2.0

First signed trust release for direct macOS distribution outside the Mac App Store.

## Highlights

- Developer ID signed and Apple notarized release assets, gated by `dist/Ryddi-release-manifest.txt`.
- Trust cockpit in the app and `reclaimer trust --json` for disk pressure, scan coverage, plan/receipt state, automation status, and release trust evidence.
- Safer cleanup planning with typed plan-condition gates, recursive open-handle checks, and final revalidation immediately before any perform action.
- General Mac cleanup surfaces for Downloads, Trash, Apps & Leftovers, Browser Caches, Package Caches, Project Dependencies, Xcode, Device Backups, and AI-agent storage.
- Remote Targets preview for agentless SSH/VPS evidence collection, redacted reports, local audit history, and manual native guidance.
- Dogfood reports and Markdown exports for shareable, local-first evidence packages.

## Install

Download the release assets from GitHub:

- `Ryddi-v0.2.0.zip`
- `Ryddi-v0.2.0.zip.sha256`
- `Ryddi-release-manifest.txt`

Verify the checksum and trust manifest before installing:

```bash
shasum -a 256 -c Ryddi-v0.2.0.zip.sha256
grep -E 'source_commit|notarization_status|stapled|gatekeeper|codesign_verified' Ryddi-release-manifest.txt
```

The expected trusted manifest state is Developer ID signing, `notarization_status=Accepted`, `stapled=true`, `gatekeeper=accepted`, and `codesign_verified=true`.

## Non-claims

- Packaging, trust readiness, dogfood reports, scans, and scheduled automation do not perform cleanup.
- Full Disk Access remains user controlled and must be granted in macOS System Settings.
- Free-space and APFS reclaim values are conservative estimates, not exact promises.
- Remote Targets v1 is report-only and does not run remote cleanup, Docker prune/reset, `rm`, sudo cleanup, or unattended destructive work.
- VM/container disks, browser profiles, GarageBand/Logic assets, Codex memories, Codex sessions, credentials, unknown app state, and user documents remain protected by default.
