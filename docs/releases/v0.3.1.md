# Ryddi v0.3.1

Trust-correctness patch for direct macOS distribution outside the Mac App Store.

Publish this release only from the exact `v0.3.1` tag after `RYDDI_VERSION=0.3.1 RYDDI_BUILD_NUMBER=4 RYDDI_RELEASE_SIGNING=required RYDDI_REQUIRE_PACKAGED_AX_E2E=1 RYDDI_ARTIFACT_BASENAME=Ryddi-v0.3.1 Scripts/release-check.sh` passes. The release manifest must prove the source commit, Developer ID signing, Apple notarization `Accepted`, stapling, Gatekeeper acceptance, strict app and nested-CLI codesign verification, packaged Accessibility E2E, bundle version `0.3.1`, and build `4`.

## Highlights

- Enforces bounded measurement depth, fair cross-root budgeting, one bounded filesystem walk, and truthful coverage evidence.
- Propagates scan cancellation into filesystem work and rejects stale completions; packaged Accessibility proof observes progress, cancellation-to-idle without a late result, and a following successful scan.
- Reconciles completed Trash actions immediately so removed rows disappear and Verify Cleanup becomes the next primary action.
- Builds one immutable audit snapshot off the main actor and preserves partial-read warnings instead of silently treating unreadable history as empty.
- Replaces inferred Full Disk Access state with operation-specific directory-listing and file-open evidence, plus a fresh permission recheck path.
- Recognizes literal, bracketed-port, and hashed SSH known-host entries through bounded system `ssh-keygen` inspection while retaining strict host-key checking.
- Converts CLI JSON encoding failures into normal nonzero errors instead of force-unwrapping.
- Signs the nested `reclaimer` executable explicitly before the app, signs the app last, and keeps deep verification without deep signing.
- Uses unique release identity `v0.3.1 (4)` across the bundle, embedded build metadata, manifest, artifact, checksum, tag, and installed-app readback.

## Install

Download these assets from the GitHub release only after its signed workflow is green:

- `Ryddi-v0.3.1.zip`
- `Ryddi-v0.3.1.zip.sha256`
- `Ryddi-release-manifest.txt`

Verify before installing:

```bash
shasum -a 256 -c Ryddi-v0.3.1.zip.sha256
grep -E 'source_commit|version|build|notarization_status|stapled|gatekeeper|codesign_verified' Ryddi-release-manifest.txt
```

Expected trusted evidence: version `0.3.1`, build `4`, `source_dirty=false`, Developer ID Application signing, `notarization_status=Accepted`, `stapled=true`, `gatekeeper=accepted`, `codesign_verified=true`, and `packaged_ax_e2e=passed`.

## Non-claims

- Unsigned local builds and developer-preview artifacts are not releases and do not prove signing, notarization, stapling, or Gatekeeper acceptance.
- Packaging, release checks, scans, reports, issue packages, and scheduled automation do not perform cleanup.
- Full Disk Access remains user controlled; Ryddi reports observed operations and does not claim to read the macOS privacy toggle.
- APFS, clone, snapshot, purgeable-space, and immediate free-space effects remain estimates until observed after an action.
- Remote Targets remain report-only and never run remote cleanup, `rm`, sudo cleanup, Docker prune/reset, or unattended destructive SSH maintenance.
- VM/container disks, browser profiles, GarageBand/Logic assets, AI-agent sessions, memories, credentials, unknown app state, and user documents remain protected by default.

`v0.3.0` remains the published trust-to-action release. `v0.4.0` is the planned guided-cleanup milestone.
