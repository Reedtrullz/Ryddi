# Ryddi v0.3.0

Trust-to-action polish release for direct macOS distribution outside the Mac App Store.

Publish this release only after `RYDDI_RELEASE_SIGNING=required RYDDI_ARTIFACT_BASENAME=Ryddi-v0.3.0 Scripts/release-check.sh` passes and `dist/Ryddi-release-manifest.txt` reports Developer ID signing, Apple notarization `Accepted`, stapling, Gatekeeper acceptance, strict codesign verification, bundle version `0.3.0`, and build `3`.

## Highlights

- Release assets use `Ryddi-v0.3.0.zip`, `Ryddi-v0.3.0.zip.sha256`, and typed release-trust manifest proof.
- Trust-to-action app surfaces for Summary, Review Queues, Package Cache, AI Agent Storage, and Remote Targets evidence.
- Signed release workflow and local signing doctor point at the current v0.3 artifact identity.
- Report-only remote target evidence, history, dogfood reports, and issue-package exports remain non-destructive.
- Redacted reports and issue packages improve review without copying raw SSH config, secrets, private keys, tokens, or arbitrary audit JSON.

## Install

Download the release assets from GitHub:

- `Ryddi-v0.3.0.zip`
- `Ryddi-v0.3.0.zip.sha256`
- `Ryddi-release-manifest.txt`

Verify the checksum and trust manifest before installing:

```bash
shasum -a 256 -c Ryddi-v0.3.0.zip.sha256
grep -E 'source_commit|version|build|notarization_status|stapled|gatekeeper|codesign_verified' Ryddi-release-manifest.txt
```

The expected trusted manifest state is bundle version `0.3.0`, build `3`, Developer ID signing, `notarization_status=Accepted`, `stapled=true`, `gatekeeper=accepted`, and `codesign_verified=true`.

## Non-claims

- Packaging, release-trust checks, dogfood reports, issue packages, scans, and scheduled automation do not perform cleanup.
- Full Disk Access remains user controlled and must be granted in macOS System Settings.
- Free-space and APFS reclaim values are conservative estimates, not exact promises.
- Remote Targets do not run remote cleanup, Docker prune/reset, `rm`, sudo cleanup, or unattended destructive SSH maintenance.
- VM/container disks, browser profiles, GarageBand/Logic assets, AI-agent memories, AI-agent sessions, credentials, unknown app state, and user documents remain protected by default.
