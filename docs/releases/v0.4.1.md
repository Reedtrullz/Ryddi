# Ryddi v0.4.1

Reclaim-guidance and cloud-footprint update for direct macOS distribution outside the Mac App Store.

Publish this release only from the exact `v0.4.1` tag after `RYDDI_VERSION=0.4.1 RYDDI_BUILD_NUMBER=6 RYDDI_RELEASE_SIGNING=required RYDDI_REQUIRE_PACKAGED_AX_E2E=1 RYDDI_ARTIFACT_BASENAME=Ryddi-v0.4.1 Scripts/release-check.sh` passes. The release manifest must bind the source commit, bundle version `0.4.1`, build `6`, signed application and nested CLI, notarization, stapling, Gatekeeper acceptance, packaged app proof, archive, and checksum.

## Candidate highlights

- Makes accepted scan results visible without presenting persistence finalization as a cancellable scan.
- Promotes one current recoverable Finder Trash scope as a conditional reclaim estimate, then opens an empty review with selected-byte, dry-run, confirmation, and verification steps.
- Routes Docker and Colima findings to their native container workspace and keeps direct-delete cache, native-tool, personal, conditional, and protected bytes outside the Finder Trash headline.
- Opens every broad or scoped cleanup review with nothing selected and prevents hidden selections from entering a plan.
- Moves container, cloud-footprint, and other operational reviews into visible Explore workspaces with local progress and error state.
- Adds explicit Dropbox, Google Drive, and selected-MEGA local-root setup guidance without claiming remote account organization.
- Adds bounded, cancellable local cloud inventory with logical-versus-allocated totals and review-only largest/oldest files.
- Adds an explicit confirmed Docker build-cache reclaim path while preserving containers, images, volumes, Colima profiles, and VM disks.
- Binds Docker inspection and pruning to the same validated context and makes scan-session persistence monotonic so older finalization cannot overwrite newer review evidence.
- Includes the Sparkle updater for signed releases after v0.4.0, with signed archive/feed verification and explicit user-controlled installation.

## Install

Download these assets only after the release manifest proves the signed gate:

- `Ryddi-v0.4.1.zip`
- `Ryddi-v0.4.1.zip.sha256`
- `Ryddi-v0.4.1-update.zip`
- `Ryddi-v0.4.1-update.zip.sha256`
- `Ryddi-release-manifest.txt`

Verify before installing:

```bash
shasum -a 256 -c Ryddi-v0.4.1.zip.sha256
grep -E 'source_commit|version|build|notarization_status|stapled|gatekeeper|codesign_verified' Ryddi-release-manifest.txt
```

Expected trusted evidence: version `0.4.1`, build `6`, `source_dirty=false`, Developer ID Application signing, `notarization_status=Accepted`, `stapled=true`, `gatekeeper=accepted`, and `codesign_verified=true`.

## Non-claims

- Dropbox, Google Drive, and MEGA support inventories confirmed local sync roots; it does not sign into, move, rename, upload, download, deduplicate, or delete remote objects.
- Docker decides which unused build-cache records are removed; Ryddi does not delete raw VM disks, containers, images, volumes, or Colima profiles.
- Full Disk Access remains user controlled; Ryddi reports observed filesystem operations rather than the privacy-toggle state.
- APFS clone, snapshot, purgeable-space, and immediate free-space effects remain estimates until observed after an action.
- Moving an item to Finder Trash does not itself increase free space.
- Packaging, release checks, scans, reports, and scheduled automation do not clean real user data.
