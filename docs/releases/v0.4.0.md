# Ryddi v0.4.0

Guided Map release for direct macOS distribution outside the Mac App Store.

Publish this release only from the exact `v0.4.0` tag after `RYDDI_VERSION=0.4.0 RYDDI_BUILD_NUMBER=5 RYDDI_RELEASE_SIGNING=required RYDDI_REQUIRE_PACKAGED_AX_E2E=1 RYDDI_ARTIFACT_BASENAME=Ryddi-v0.4.0 Scripts/release-check.sh` passes. The release manifest must prove the source commit, Developer ID signing, Apple notarization `Accepted`, stapling, Gatekeeper acceptance, strict app and nested-CLI codesign verification, packaged Accessibility E2E, bundle version `0.4.0`, and build `5`.

## Highlights

- Opens to a user-started Guided Map instead of beginning a scan automatically.
- Uses a deterministic proportional treemap with an equivalent accessible outline, breadcrumbs, evidence state, and inspection-only file actions.
- Keeps the regular-user navigation focused on Home, Explore, and History while preserving specialist reports in Advanced Settings.
- Shows current free space, measured allocated storage, scan coverage, and reclaim estimates as distinct claims.
- Starts every cleanup review with nothing selected; map interaction never grants cleanup authority.
- Retains Check Safely, current dry-run evidence, exact-path confirmation, one-use Finder Trash authorization, final identity/policy/symlink/open-handle checks, receipts, and a verification scan.
- Preserves the last trustworthy map across cancelled, failed, superseded, or non-critical persistence failures.
- Adds packaged Accessibility proof for cancellation without late commit, explicit selection, Trash execution, row removal, verification, protected fixtures, and responsive containment at 820x620, 1180x760, and 1440x900.
- Includes the isolated Protect foundation without linking provider or credential targets into the app or CLI.
- Uses unique release identity `v0.4.0 (5)` across the bundle, embedded build metadata, manifest, artifact, checksum, tag, and installed-app readback.

## Install

Download these assets only after the release manifest proves the signed gate:

- `Ryddi-v0.4.0.zip`
- `Ryddi-v0.4.0.zip.sha256`
- `Ryddi-release-manifest.txt`

Verify before installing:

```bash
shasum -a 256 -c Ryddi-v0.4.0.zip.sha256
grep -E 'source_commit|version|build|notarization_status|stapled|gatekeeper|codesign_verified' Ryddi-release-manifest.txt
```

Expected trusted evidence: version `0.4.0`, build `5`, `source_dirty=false`, Developer ID Application signing, `notarization_status=Accepted`, `stapled=true`, `gatekeeper=accepted`, `codesign_verified=true`, and `packaged_ax_e2e=passed`.

## Non-claims

- Automated Accessibility proof does not replace a human VoiceOver review.
- Full Disk Access remains user controlled; Ryddi reports observed operations and does not claim to read the macOS privacy toggle.
- APFS clone, snapshot, purgeable-space, and immediate free-space effects remain estimates until observed after an action.
- Moving an item to Trash does not itself increase free space.
- Map selection, drill-down, search, and filters never select or execute cleanup.
- Packaging, release checks, scans, reports, issue packages, and scheduled automation do not clean real user data.
- Remote Targets remain report-only and never run remote cleanup, `rm`, sudo cleanup, Docker prune/reset, or unattended destructive SSH maintenance.
- VM/container disks, browser profiles, creative assets, personal documents, AI-agent sessions, memories, credentials, and unknown app state remain protected by default.
